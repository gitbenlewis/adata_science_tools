When true, force the overall fit line and fit-handle creation even if `show_all_obs_fit=False`, build footer text as `No valid {subset_key} groups after filtering; showing All data fit.` followed by the normal `All data: ...` stats line, and skip subgroup-specific fit handles because none exist. Keep `show_stats_text` and `show_fit_legend` as display toggles only.
3. Legend and marginal behavior: In the same fallback branch, if `show_fit_legend=True`, render a one-entry legend for the forced overall fit with title `All data fit\n{corr_label}_corr` instead of `<subset_key> fit\n...`. 4. Documentation: Update `docs/_corr_dotplots.md` to document this dev-only exception. State that `corr_dotplot_dev()` falls back to all-data fit/stats/marginals when no valid subset groups remain, while `corr_dotplot()` intentionally keeps its current behavior.

Assert that with `show_all_obs_fit=False`, `show_x_marginal_hist=True`, `show_y_marginal_hist=True`, `show_all_obs_x_hist=False`, and `show_all_obs_y_hist=False`, the main axes still draw the all-data fit, the fit legend contains exactly one `All data` entry with the fallback title, the footer contains both the fallback note and the all-data stats line, and both marginal panels render all-data histogram artists. Add a second assertion path that `show_fit_legend=False` and `show_stats_text=False` suppress only those UI surfaces while preserving the fallback fit and fallback marginals.
6. Assumptions and constraints: Preserve public API and return values. Do not change correlation math, filtering order, or scientific/statistical behavior. Trigger the fallback only when there are no subgroup values after filtering, not when subgroup values exist but an individual subgroup fit is unavailable.
