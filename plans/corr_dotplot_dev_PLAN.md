# Plan: Add `corr_dotplot_dev()` With Optional Marginal Histograms

1. Summary: Leave `corr_dotplot()` unchanged and add a new `corr_dotplot_dev()` beside it in `_plotting/_corr_dotplots.py`, using the current `corr_dotplot()` body as the starting point. Preserve the current data assembly, filtering (`nas2zeros`, `dropna`, `dropzeros`), correlation/regression calculations, scatter styling, stats footer, and legend toggles unless marginal panels are explicitly enabled.

2. Public API: `corr_dotplot_dev()` should copy the current `corr_dotplot()` signature and behavior, plus these new kwargs: `show_x_marginal_hist: bool = False`, `show_y_marginal_hist: bool = False`, `x_marginal_hist_bins: int | Sequence[float] = 20`, `y_marginal_hist_bins: int | Sequence[float] = 20`, `show_all_obs_x_hist: bool = False`, `show_all_obs_y_hist: bool = False`, `x_marginal_hist_height_ratio: float = 0.18`, and `y_marginal_hist_width_ratio: float = 0.18`. Keep the return tuple shape at 5 values, but make the second value a dict: `{"main": ax_main, "x_marginal": ax_top_or_none, "y_marginal": ax_right_or_none}`. Do not add `spearman_cor_dotplot_dev()` in this change; `corr_dotplot_dev(method="spearman", ...)` is the supported Spearman path.

3. Layout and behavior: Use `GridSpec` only inside `corr_dotplot_dev()`: a single axes when both marginals are off, a 2x1 shared-x layout when only the x marginal is on, a 1x2 shared-y layout when only the y marginal is on, and a 2x2 layout with a hidden top-right cell when both are on. The main scatter axes remains the owner of the scatter, fits, title, axis labels, axis lines, stats text, and returned regression outputs. Marginals must always plot from the same filtered `working_df` used by the scatter and fits. If `subset_key` is provided, marginals group by `subset_key`; `hue` remains scatter-only. If `subset_key` is not provided, each enabled marginal draws one overall histogram. Use common bins per axis from all filtered values on that axis, internal `element="step"`, subgroup alpha `0.35`, and all-observation overlay color `"0.7"` with alpha `0.20`; these stay internal defaults in v1 rather than new public styling knobs. Hide x tick labels and y ticks on the top marginal, hide y tick labels on the right marginal, and keep the right marginal x-axis as counts. When a y marginal is enabled, default legends move outside the full figure right edge rather than into the gap between the scatter and histogram; explicit `fit_legend_bbox_to_anchor` and `hue_legend_bbox_to_anchor` overrides still win.

4. Implementation boundaries: Do not refactor or change the stable `corr_dotplot()` or `spearman_cor_dotplot()` behavior in this change. Implement `corr_dotplot_dev()` as a separate function by copying the current `corr_dotplot()` flow and applying the new layout and marginal logic locally, while continuing to reuse the existing helper functions already in the module. Rely on the existing wildcard re-exports so the new public function is available through `adata_science_tools` without changing package `__init__` files unless an `__all__` is introduced later. Update the docs page for `_corr_dotplots` to list and document `corr_dotplot_dev()` as the marginal-enabled experimental variant and to show the new `axes["main"]` / `axes["x_marginal"]` / `axes["y_marginal"]` return pattern.

5. Test plan: Extend `tests/test_corr_dotplots.py` with `unittest` coverage for `corr_dotplot_dev()` in all four layout modes (`none`, `x`, `y`, `both`), asserting that `axes["main"]` is always present and disabled panels are `None`. Add tests for grouped marginals with `subset_key`, all-observation overlays, missing-value filtering parity with the main scatter, custom bin counts, `show=False` behavior, footer non-overlap after layout adjustment, and legend coexistence when a y marginal is shown. Add one regression test confirming the original `corr_dotplot()` still returns a single `Axes` and preserves its current legend behavior unchanged.

6. Assumptions and defaults: Treat `corr_dotplot_dev()` as a new public but experimental API. `figsize` continues to describe the full figure including any marginal panels. User-supplied legend anchors keep their current meaning and may still overlap the new panels if callers place them there deliberately. High-cardinality `subset_key` values remain supported the same way subgroup fit lines are supported today; this change does not add automatic pruning, collapsing, or new guardrails for crowded subgroup overlays.
