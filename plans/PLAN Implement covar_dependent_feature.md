# Implement `covar_dependent_feature.py`

Summary: add a small simulation module in `_simulate_data` that 1) builds `obs_df` from per-covariate distribution specs, 2) generates feature values by vectorized linear combination of those covariates, and 3) optionally returns and saves an `AnnData` dataset by reusing the existing `save_dataset(...)` helper instead of duplicating IO code.

1. Public API: implement `sim_observations_covars(..., random_seed: int | None = None)`, `sim_covar_dependent_features(...)`, and `sim_covar_dependent_dataset(..., random_seed: int | None = None)`. `random_seed` is added only to the functions that perform random draws; `sim_covar_dependent_features(...)` stays deterministic given `obs_df`, `betas`, and `yints`.
2. Input schema: `obs_covar_dist_params` will be a dict keyed by covariate name, for example `{"Age": {"dist": "normal", "mean": 10, "stdev": 5}, "gender": {"dist": "binomial", "prob": 0.5}}`. Support `normal` and `binomial`, and accept `bionomial` as an alias for the typo in the prompt. `obs_df` columns will be exactly `obs_key_list` in order; distribution parameters will not be duplicated as columns.
3. `sim_observations_covars(...)`: coerce `n_obs` to `int`, build a 1-based index using `obs_names_prefix` (`obs_1`, `obs_2`, ...), generate each covariate column with NumPy, store normal draws as float and binomial draws as `0/1`, and optionally save `obs_df` to CSV. If `save_obs_df_path` has no suffix, append `.csv`.
4. `sim_covar_dependent_features(...)`: treat all `obs_df` columns as predictors in column order, coerce them to a numeric matrix, and compute `X` with vectorized linear algebra. Normalize parameters with these rules: a 1D `betas` vector of length `n_covars` broadcasts to every feature; a 2D `betas` array with shape `(n_vars, n_covars)` provides per-feature coefficients; scalar `yints` broadcasts to all features; a 1D `yints` sequence must have length `n_vars`. Build `var_df` with index `var_names` and columns `yint` plus one column per predictor named `beta_<obs_key>`.
5. AnnData/save behavior: create `adata = ad.AnnData(X=X, obs=obs_df.copy(), var=var_df.copy())` whenever `also_return_adata=True` or `save_adata_dataset=True`. Reuse `.._io._IO.save_dataset` via a local `_save_dataset` alias. If `save_adata_dataset=True` and `output_path is None`, save to `Path.cwd() / "covar_dependent_dataset"`. If `also_return_adata=False` and saving is disabled, return `adata=None` in the 4-tuple.
6. Wrapper behavior: `sim_covar_dependent_dataset(...)` should expose the explicit union of the lower-level arguments, call `sim_observations_covars(...)` first, then pass the resulting `obs_df` into `sim_covar_dependent_features(...)`, and return the same `(X, var_df, obs_df, adata)` tuple. Keep the wrapper flat and explicit rather than using `**kwargs`.
7. Validation and failure modes: raise clear `ValueError` or `TypeError` for empty `obs_key_list`, missing covariate specs, unsupported distributions, invalid `prob` or `stdev`, empty `obs_df`, non-numeric/non-coercible predictor columns, duplicate `var_names`, and shape mismatches between `obs_df`, `betas`, `yints`, and `var_names`. Keep validation local and minimal; no extra abstraction layer.
8. Package integration: add `__all__` in `covar_dependent_feature.py` and update `_simulate_data/__init__.py` to export the three functions. Do not update package-root `adata_science_tools.__init__.py` in v1; keep this scoped to the `_simulate_data` subpackage unless broader export is requested later.
9. Tests: add `tests/test_simulate_data.py` covering seeded reproducibility for `sim_observations_covars(...)`, exact `X` values for deterministic feature generation, broadcasted `betas` and `yints`, wrapper integration, CSV and h5ad sidecar creation in a temp directory, and representative validation errors.
10. Assumptions/defaults: v1 includes no noise term in `sim_covar_dependent_features(...)`; simulated features are purely linear covariate-dependent values. Binomial means one Bernoulli trial per observation. The implementation should stay NumPy/pandas vectorized, use no new dependencies, and reuse the existing dataset-save helper rather than copying the pasted `_save_dataset(...)` body.
