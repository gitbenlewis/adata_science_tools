# Add Residual `y` Variance to `covar_dependent_feature.py`

Summary: extend the covariate-dependent feature simulator so observed feature values can include configurable residual variance around the current linear mean model. The goal is to preserve the existing linear signal while making simulated points look like realistic noisy observations instead of lying exactly on a line.

1. Objective: modify [`_simulate_data/covar_dependent_feature.py`](../_simulate_data/covar_dependent_feature.py) so residual `y` variance is part of the formal simulation API rather than a script-level postprocessing step. The current deterministic behavior should remain the default to preserve backward compatibility.
2. Public API changes: update `sim_covar_dependent_features(...)` and `sim_covar_dependent_dataset(...)` to accept residual-noise arguments. The minimal parameter set should be `residual_dist: str = "normal"`, `residual_mean: float = 0.0`, `residual_stdev: float = 0.0`, and `random_seed: int | None = None`. `sim_observations_covars(...)` does not need residual-noise arguments because it only generates covariates.
3. Backward compatibility: keep `residual_stdev=0.0` as the default so existing code paths continue to return the same noiseless values they return today. This preserves current behavior unless callers explicitly opt into residual variance.
4. Statistical model: keep the current linear mean structure unchanged. The updated model should be `X_mean = obs_matrix @ beta_matrix.T + yint`, followed by `X = X_mean + residuals`, where `residuals` are sampled independently for each observation-feature pair. In v1, support only additive normal residuals. Do not change the coefficient interpretation, intercept interpretation, or covariate simulation behavior.
5. Residual generation details: implement residual simulation inside `sim_covar_dependent_features(...)` using NumPy and a local random generator seeded from `random_seed`. `residual_mean` and `residual_stdev` should support scalar broadcasting across all features. A later extension could support per-feature residual parameters, but that is out of scope for this first pass unless there is a strong reason to add it now.
6. Validation and failure modes: reject unsupported `residual_dist` values with a clear `ValueError`, reject negative `residual_stdev`, and preserve the current beta, intercept, and predictor-shape validation. Keep validation local and minimal; do not add a separate parameter-normalization abstraction unless the change becomes materially more complex.
7. Output metadata: extend `var_df` to record the residual settings used for each simulated feature, for example `residual_dist`, `residual_mean`, and `residual_stdev`. When `adata` is returned, store the noiseless linear predictor in `adata.layers["linear_mean"]` and the realized residual values in `adata.layers["residual"]` so downstream users can inspect the signal and noise components separately without changing the meaning of `adata.X`.
8. Wrapper behavior: pass the new residual arguments through `sim_covar_dependent_dataset(...)` explicitly, keeping the wrapper flat and readable like the current implementation. Do not switch to `**kwargs`.
9. Tests: update [`tests/test_simulate_data.py`](../tests/test_simulate_data.py) to cover zero-noise backward compatibility, seeded reproducibility when `residual_stdev > 0`, correct `var_df` residual metadata, correct `adata.layers["linear_mean"]` and `adata.layers["residual"]` contents, wrapper passthrough of residual arguments, and representative validation failures for invalid residual settings.
10. Documentation: update [`docs/_simulate_data.md`](../docs/_simulate_data.md) and any related README text that currently describes the simulator as purely deterministic. The docs should explain that the mean model is still linear and deterministic, but the observed simulated feature can now include additive residual noise when requested.
11. Example follow-on: after the library change lands, update [`example_simulated_data/scripts/simulate_1_var_covar_age.py`](../example_simulated_data/scripts/simulate_1_var_covar_age.py) and [`example_simulated_data/config/config.yaml`](../example_simulated_data/config/config.yaml) to pass config-driven `residual_mean` and `residual_stdev` into the library. That follow-on change should remove any need for script-local noise injection and keep the example aligned with the corrected library API.
12. Non-goals for this pass: do not add heteroskedastic residuals, do not add non-normal residual families beyond a validated placeholder error path, do not change the package-root exports, and do not refactor unrelated simulation code.
